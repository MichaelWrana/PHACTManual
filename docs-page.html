Introduction Last updated: 2019-06-01

Welcome to the PHACT User Manual. This user guide is designed to provide documentation for people who will use PHACT on a day-to-day basis. This document is designed to be read by any user of PHACT, as most users will have access to the features documented here. Site Administrator functionalities are covered in a separate section of this guide.

What is PHACT?

Capturing anonymous, publicly-available data, we track the location pathways and interactions of individuals within a public environment. We call this Passive Historical Aggregate Contact Tracing, or PHACT. PHACT data is anonymous and publicly available, so opt-in by the individual is not required, eliminating privacy concerns. PHACT information is available through a SAAS platform and to the public through a mobile app. PHACT data is accurate and comprehensive. PHACT tracks the location pathways and interactions of all individuals within a public environment in real time, giving insight into exposure events and geo-specific hotspots. ​ Traditional peer-to-peer contact tracing applications provide basic information on exposure between individuals. PHACT nano-granular data aggregation extends analytics to include:

  • The level and speed of contagion
  • The differentiation of emerging variants
  • Identification of emerging hot-spots
  • Predictive mitigation strategies

User Access

Gain access to the PHACT Web portal using your account. View personal account information and access administrator features. Signing off and switching accounts.

Login

  1. Navigate to PHACT web page (phact.ca)
  2. Locate and press phact login button
  3. Allow phact portal homepage to load
  4. Enter username and password in appropriate sections
  5. Enter 2FA Code (Currently disabled)

Logout as User

  1. Click the icon in the top-right corner of the screen to open a drop-down menu
  2. Click sign-out from the drop down menu
  3. Select “OK” in the pop-up confirmation menu that will appear in the center of your screen

Logout as Admin

  1. Hover your mouse over the very top of the PHACT website until the admin menu bar appears
  2. Move to the top-right corner and click the exit icon that appears above the user icon (DO NOT click the ‘x’)
  3. Select “OK” in the pop-up confirmation menu that will appear in the center of your screen

View Account Details

  1. Click the icon in the top-right corner of the screen to open a drop-down menu
  2. Your currently logged in account information will appear at the top of the drop down menu (Example: Signed in as: Administrator)

View Administrator Panel

  1. Hover your mouse over the very top of the PHACT website until the admin menu bar appears
  2. Select “ADMIN>>” from the top-left corner of the screen
  3. The Administrator panel will open in a new tab in your browser

Dashboard

The Homepage of the PHACT Portal. View a heatmap of the target environment. See detailed information about each installed node. See environment capacity gauges.

Capacity Gauges

Capacity gauges for determining how congested each zone of the monitored environment as well as an overall capacity gauge.

Environment Heatmap

  • At the bottom of the page there is a live map of the environment which indicates the name and location of each individual node.
  • These individual nodes can flash green, yellow, or red, depending on how many devices it detects in it’s range.
  • Each node’s sensitivity, can be adjusted in node settings
  • Use the dashboard map to familiarize / remind yourself of the location of nodes when responding to incident reports.

Node Details

Each node can be clicked on to open a detailed view of the node’s settings and activity.

History

Overview of historical recorded information from each node in the target environment. See graphs representing number of contacts near each node.

View Node History

  • At the top of the landing page of the phact website, the second heading from the left is “History”.
  • This is a graph of the “contacts” in the area of each node within an interval specified in the device settings.
  • Sideways colored bars indicate when a zone is over-populated.
  • Each Dot can be hovered over with your mouse to show the number of detected devices at a specific timestamp.

Incidents

See all incidents that have occured in the target enviornment. Respond to an incident from a computer terminal and assign incidents to users.

Respond to an Incident as a Warden

  1. You receive an alert from the PHACT app that a node is overpopulated.
  2. Navigate to “Incidents” to identify the job type and description.
  3. You will see a list of active, in-progress, and non-assigned jobs
  4. Locate the job code that matches the alert you received. (You can search your job code in the top left search bar)
  5. Adjust the orange “RESPOND” box
    • Select yourself as the user by clicking the “Select a User” text.
    • Click on the text “Assignment” and adjust the status of the job you are responding to, the options are as follows: waiting, pending, in progress, need assistance, escalated, revise, reject, review, complete, approve, not required.
    • The PHACT user will determine which of these statuses will be utilized.
  6. Locate the node on the “dashboard” map and respond to the social distancing issue
  7. Once the people at the node are socially distancing, navigate back to the “incidents” tab and change the orange “RESPOND” box from “in progress” -> “complete”.

Respond to an Incident from Mobile Device

Approving an Incident Report as Front Desk

  • You will receive an alert from the phact app when a new job comes in
    • Generally let the Wardens self-assign tasks and manage the tasks themselves.
    • Once a Warden has responded to a task, and reported the incident as “in progress” the front desk user will navigate to the “incidents” page and then monitor the “in-progress” tasks.
    • A Warden / security user has several ways they can tag a social distancing event which have different implications for a front desk user.
Waiting A front desk user would set a task to “waiting” if it hasn’t been picked up by a Warden user.
Pending A Warden user has accepted the job, but is in progress of another job. If the problem becomes urgent, the front desk user could re-assign the job to a different warden user
In Progress A warden is currently addressing the socially distancing issue.
Need Assistance A front desk user would notify other warden users that help is needed at a specific node / location.
Escalated If a Warden has set a job as complete, but the front desk user believes there is still an issue at the node. The front desk user can set the task to “Revise” to notify the warden to return to the situation.
Reject If a flight is boarding, and the passengers are already socially distant, then a Warden can reject the job as nothing can be done. For a front desk user, this might imply working with admin to adjust the sensitivity of a certain node to prevent redundant alerts.
Complete A Warden has completed a task, and now requires approval for the job to be finished. Check the node that the job was for and see if the issue has be resolved.
Approve After checking the node, change the orange box where the warden has assigned himself to the task, and change the status to “Approved”. This will delete the job from the “incidents” page as it’s been completed.
Not Required A warden responded to a triggered event at a node, and there was no issue when he arrived.

Contact Tracing

Enter a device in the target environment and a timeframe to view a history of the other devices they have interacted with. Notify relevant parties of potential infection.

View Device History

  1. Enter the MAC Address of a specific device that was in the environment.
  2. Enter the 2 week period with which you wish to view.
  3. Select “Search”
  4. The total number of contacts this device has interacted with will appear along with a graph indicating contacts during the specified search period.
  5. Hovering over the graph with your mouse will show the number of other devices this one has interacted with at a specific timestamp.
  6. Below the chart, the list of each node this device has interacted with is listed.

Save Device History

  1. Click on the node history list (See View a Device’s Contact History)
  2. The entire list can be copied and pasted to any file you choose (command/control + A to select the entire list for copying).

Manage Users

Create new user accounts and remove existing ones. Control user permissions and website access. View a user's incident reporting history.

Add a New User

Warning

The user must be assigned to a group or they will be unable to access the dashboard. See: Change a User's Groups.
This user information cannot be changed later.

  1. Select “Add User” in the top-right corner to open the add user pop-up menu.
  2. Fill in the user’s profile picture, login, name, and password.
  3. Select “Add” at the bottom of the pop-up to create a new user.

Remove an Existing User

Note

Removing dependencies assoicated with a user will also remove their task history and assignments. If you would like to create a new user with the same information but retain task history, DO NOT fill the checkboxes. If you would like to create a new user with the same name but different history, fill the checkboxes.

  1. Find the user you would like to remove from the manage users list
  2. Select the trash bin icon on the far right. This will open the delete user pop-up menu.
  3. OPTIONAL: Click the checkboxes for any dependencies associated with the user.
  4. Select the red “delete” button at the bottom of the pop-up.

Change a User's Groups

  1. Find the user you would like to update from the manage users list.
  2. Select the user group assignment button on the right side. This will open the user group assignment pop-up menu.
  3. Select the checkboxes for each role this user should be assigned to. Each user can be assigned to multiple groups.
  4. Select the blue “Save” button near the top right of the pop-up menu.

View a User's Details

  1. Find the user you would like to investigate from the manage users list.
  2. Select them by clicking the profile picture or login (underlined) from the list. This will open the user details page.
  3. Here, you can view a user’s code, department, total number of tasks, task history, and currently assigned tasks.

View a User's Task History

  1. From the User Details menu, click the “3 Month Cale...:” button on the bottom bar.
  2. This will display a calendar with any in progress or completed jobs listed during each day.
  3. Select on a specific day to view a listing of every task that began, was in progress, or completed on that date.

View a User's Assigned Tasks

  1. From the User Details menu, click the “Tasks” button on the bottom bar.
  2. This will show a list of all tasks currently not complete that are assigned to the selected user. A timetable of these tasks can be found on the right side

Node Settings

View and customize the settings associated with each node in the environment. Customize the breakpoints for creating an incident. Add detailed notes for each node.

Adjust Node Sensitivity

Note

Sensitivity setting changes may not immediately update on the Dashboard and Incidents panel. Wait 10 minutes before attempting to change the senstivity settings on the node again.

  1. Find the device from the devices list. The “Zone” folders may help quickly find the correct node.
  2. Left-click the green, yellow, or red box in the device’s row to edit the sensitivity setting. This represents the number of devices allowed within the node's radius.
    • Green: indicates devices that are always in the node's radius (i.e. TV Screen, Camera, etc.)
    • Orange: indicates the maximum number of allowed people based on social distancing protocos.
    • Red: indicates the device is over-capacity and too many people are present. An Incident will be created.
  3. Click the “enter” key to save the setting that has been changed.
  4. Multiple device settings can be modified at once. A modified setting will be highlighted green.
  5. Click the “save” button on the top right corner of the page to save all sensitivity changes at once.

Add a Note to a Device

  1. Find the device which you would like to add a note to from the devices list. The “Zone” folders may help in quickly finding the correct node.
  2. If the devices has existing notes, click the “Notes” button on the right side of the device’s row.
    1. This will list any existing notes associated with the specific device.
    2. Select the “+” icon in the top right to add a new note to the same device.
    3. Follow the instructions for “no existing notes” below.
  3. If the device has no existing notes, click the “no notes” button on the right side of the device’s row.

View a Device's Notes

  1. Find the device which you would like to add a note to from the devices list. The “Zone” folders may help quickly finding the correct node.
  2. Select the “Notes” button on the right side of the device’s row. The notes will appear as a sidebar.

Delete an Existing Note

  1. Find the note you would like to delete (see View a Device’s Notes)
  2. From the list on the sidebar, select the ‘edit’ button to open the actions drop-down menu.
  3. Select “Delete Note”.
  4. Select “OK” in the confirmation pop-up menu.

Modify an Existing Note

Note

Already created notes cannot be mailed to users or have attachments added. To utilize this functionality a new note must be created. See Add a Note to a Device

  1. Find the note you would like to view (see View a Device’s Notes)
  2. From the list on the sidebar, click the ‘edit’ button to open the actions drop-down menu.
  3. Select “Edit Note”.
  4. Modify the note’s contents from the pop-up menu.
  5. Select “Save” to save the changes to the note.

Change the Status of a Note

  1. Find the note you would like to change (see View a Device’s Notes)
  2. From the list on the sidebar, select the ‘edit’ button to open the actions drop-down menu.
  3. Select “Edit Status” to open the note status pop-up menu.
  4. Select “-Select-” to open the drop-down menu of possible note statuses. Instead of ‘-Select-” it may instead list the current status of the note
  5. Select the desired status for the note from the drop-down menu.
  6. Select “Save”.

PHACT Mobile App

Downloading and accessing the associated PHACT mobile app. Remotely updating the status of an incident and requesting assistance.

Downloading the PHACT App

Unknown

Login

  1. Open the PHACT App on your phone (If you do not have the app see Downloading the PHACT App)
  2. Enter your username and password in the text prompt that appears.
  3. Select Login.

Logout

  1. Open the side navigation menu by clicking the three lines icon.
  2. Select Sign In.
  3. Select Log Out.

View Assigned Tasks

  1. Open the side navigation menu by clicking the three lines icon.
  2. Select My Tasks.
  3. Select SEARCH to update the list with any new tasks.

Update Task Status

  1. Navigate to the task list (see Viewing Assigned Tasks)
  2. Identify the task you want to update the status of.
  3. Select either escalate or complete. The definitions for these statuses can be found under Approve an Incident Report
  4. Select SEARCH to update the list.

Configure Push Notifications

  1. Open the side navigation menu by clicking the three lines icon.
  2. Select Notifications.
  3. Select Register for Push.
  4. A pop-up window from your device may appear asking for access to send push notifications.
  5. Select Yes.

View App Details

  1. Open the side navigation menu by clicking the three lines icon.
  2. Select About.
